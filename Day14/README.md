Weekly Review and Reflection :

Objective:
The goal of this task was to reflect on the concepts learned during the previous cybersecurity exercises and understand how they connect in a real-world security workflow.

1. Concept That Became Clearer:

One concept that became much clearer to me is network traffic analysis using Wireshark. By observing live packets from my own system, I understood how data travels between devices and servers. I was able to see protocols like DNS, TCP, and HTTP in real time and understand how different services communicate over the network.

2. Topic That Is Still Difficult

A topic that I still find slightly difficult is interpreting complex packet data in Wireshark. While identifying basic protocols is easy, understanding every field inside packet details (such as flags, sequence numbers, and handshake processes) requires more practice and deeper networking knowledge.

3. Improvement in My Learning Process:

One improvement in my cybersecurity learning process is that I have started combining theory with hands-on tools. Instead of only reading about concepts, I now test them using tools like Nmap, Wireshark, and VPN services, which helps me understand how cybersecurity works in real environments.

4. Connection Between Two Tasks

I noticed a strong connection between Nmap scanning (Day 9) and Firewall basics (Day 10).

Nmap helps discover open ports and services running on a system, while firewalls are used to control or block access to those ports. This means once open services are discovered through scanning, firewall rules can be configured to protect or restrict access to them.

My Key Learning Summary:

The tasks from Day 8 to Day 13 together represent a basic cybersecurity workflow:

1. Protect accounts using strong passwords
2. Discover open ports using Nmap
3. Understand running network services
4. Protect systems using firewalls
5. Hide or protect identity using VPN
6. Monitor real network traffic using Wireshark

This workflow demonstrates how different security concepts connect together when analyzing and securing a system.

Conclusion:

This week helped me understand how different networking and security tools interact with each other. Practicing these tasks improved my practical understanding of cybersecurity and helped me think more like a security analyst when examining systems and networks.
